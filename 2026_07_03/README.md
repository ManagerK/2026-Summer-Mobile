# 📅 Naver Calendar Clone Project (네이버 캘린더 클론 코딩)

본 프로젝트는 대학교 **앱프로그래밍** 과목의 실습 과제로 진행되는 캘린더 애플리케이션 개발 프로젝트입니다. 기존의 '네이버 캘린더' 앱의 핵심 기능을 모방(Clone)하여 프론트엔드 앱 로직을 구현하고, 수집된 데이터를 **Datalore** 환경과 연동하여 분석하는 것을 목표로 합니다.

---

## 📱 1. 모방 대상 앱 소개
* **대상 앱:** 네이버 캘린더 (Naver Calendar)
* **핵심 타겟 기능:** * 월간 달력 View 인터페이스 및 UI 레이아웃
  * 일정 추가/수정/삭제 (CRUD) 및 카테고리 태깅 기능
  * 날짜별 일정 리스트 조회 및 데이터 로컬 저장

---

## 🎯 2. 앱 모방 배경 및 선정 이유 (Motivation)

1. **기능의 직관성과 친숙함**
   * 일상생활에서 가장 대중적으로 사용되는 앱 중 하나로, 요구사항 분석과 UI/UX 설계 단계에서 기획의 모호함을 줄이고 개발 본연의 로직에 집중할 수 있어 첫 클론 과제로 적합하다고 판단했습니다.
2. **앱프로그래밍 기술 스택의 체계적 훈련**
   * 안드로이드 스튜디오의 핵심 컴포넌트인 `CalendarView`, 일정 목록을 유연하게 표현할 수 있는 `RecyclerView`, 화면 간 데이터 전달(`Intent`), 그리고 데이터 보존을 위한 로컬 데이터베이스(`Room` 또는 `SQLite`)까지 앱 개발의 필수 요소를 종합적으로 학습할 수 있습니다.
3. **Datalore 연동 및 데이터 분석 시너지 (AI 융합 전공 연계)**
   * 사용자가 등록한 일정 데이터(시간, 카테고리, 빈도 등)는 훌륭한 분석 소스가 됩니다. 이를 Datalore 파이썬 환경으로 연동하여 사용자의 시간 투자 비율(원형 차트), 요일별 바쁨 지수(막대그래프) 등 '개인화된 시간 관리 리포트'를 도출하기에 가장 최적화된 주제이기 때문에 선정했습니다.

---

## 🗺️ 3. 개발 계획 및 로드맵 (Roadmap)

본 프로젝트는 설계부터 데이터 분석 연동까지 총 4단계에 걸쳐 순차적으로 진행됩니다.

### 📌 Phase 1: 개발 환경 구축 및 UI 레이아웃 설계
* 안드로이드 스튜디오 프로젝트 초기화 및 기본 환경 설정
* 메인 화면 구성: `CalendarView`를 활용한 월간 달력 레이아웃 구현
* 일정 등록 화면 구성: 일정 제목, 시간 설정, 카테고리(공부, 운동, 휴식 등) 선택 스피너(`Spinner`) 디자인

### 📌 Phase 2: 데이터베이스 연단 및 로컬 CRUD 구현
* 로컬 데이터베이스 데이터 모델(Entity) 및 데이터 접근 객체(DAO) 설계
* 일정을 데이터베이스에 저장하고, 달력에서 특정 날짜를 선택했을 때 해당 날짜의 일정을 `RecyclerView` 리스트로 불러오는 연동 로직 완성

### 📌 Phase 3: Datalore 분석 환경 연동 및 데이터 파이프라인 구축
* 로컬 DB에 축적된 일정 데이터를 CSV 또는 JSON 형태로 추출하는 기능 구현
* Datalore(Jupyter 환경)로 데이터를 전달하는 연동 구조 확인 및 데이터프레임(Pandas) 로드 실습

### 📌 Phase 4: 시각화 리포트 구현 및 최종 고도화
* Datalore 파이썬 라이브러리(Matplotlib / Seaborn)를 활용한 카테고리별 시간 소비 리포트 시각화 완성
* 앱 UI 디테일 정돈, 예외 처리 테스트 및 최종 과제 산출물 레포트 작성
