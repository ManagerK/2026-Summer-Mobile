# 📅 Naver Calendar Plus Project (네이버 캘린더 UI/UX 및 기능 개선 클론 코딩)

본 프로젝트는 대학교 **모바일 프로그래밍** 과목의 실습 과제로 진행되는 캘린더 애플리케이션 개발 프로젝트입니다. 기존 '네이버 캘린더' 앱의 핵심 인터페이스를 모방(Clone)하여 구현하되, 기존 앱이 가진 일정 등록의 번거로움과 시각적 가시성의 한계를 보완하여 **빠른 일정 추가(Quick Adding)** 및 **직관적인 내부 시각화 시스템**을 갖춘 사용자 중심의 캘린더로 개선하는 것을 목표로 합니다.

---

## 📱 1. 대상 앱 소개 및 개선 방향
* **대상 앱:** 네이버 캘린더 (Naver Calendar)
* **핵심 타겟 기능:**
  * 월간 달력 View 인터페이스 및 UI 레이아웃
  * 일정 추가/수정/삭제 (CRUD) 및 로컬 저장 (`Room` 라이브러리 활용)
  * 날짜별 일정 리스트 조회 및 카테고리 태깅 기능
* **주요 개선 항목:**
  1. **초고속 일정 추가 (Quick Add UI):** 기존 앱의 복잡한 등록 절차를 탈피, 달력 내 특정 날짜를 '롱 클릭(Long Click)' 하거나 '드래그(Drag)'하여 팝업 레이어 형태로 일정을 즉시 추가하는 간편 인터페이스 도입
  2. **카테고리 컬러 태깅 및 인앱 시각화:** 달력 메인 View에서 사용자가 지정한 카테고리별 고유 색상이 미니 도트(Dot) 형태로 표현되도록 하여, 별도의 메뉴 이동 없이 월간일정 분포를 직관적으로 파악할 수 있도록 UI 개선
  3. **중요 마감 일정(D-Day) 하이라이트:** 일반 일정과 마감일(시험, 과제 등)의 데이터 타입을 분리하고, 당일 일정 리스트(`RecyclerView`) 상단에 고정 및 잔여 시간 디스플레이 적용

## 🎯 2. 앱 선정 이유 및 개선 배경 (Motivation)

1. **기능의 직관성과 UI/UX 개선의 필요성 (Pain Point 분석)**
   * 네이버 캘린더는 대중적인 앱이지만, 일정을 하나 등록하기 위해 매번 새로운 전체 화면으로 이동하여 제목, 시간, 알림 등을 일일이 설정해야 하는 뎁스(Depth)의 번거로움이 있었습니다. 또한, 수많은 일정이 단순 텍스트 형태로만 나열되어 있어 이번 달에 내가 어떤 종류의 일정(학업, 운동, 휴식 등)에 시간을 많이 투자했는지 한눈에 인지하기 어렵다는 아쉬움이 존재했습니다.
2. **모바일 프로그래밍 기술 스택의 체계적 훈련**
   * 안드로이드 스튜디오의 핵심 컴포넌트인 `CalendarView` 커스텀 및 터치 이벤트(`GestureDetector`) 처리, 일정 목록을 유연하게 표현하고 가시성을 높이는 `RecyclerView`, 다이얼로그 플래그먼트(`DialogFragment`)를 활용한 팝업 UI 구현, 그리고 데이터 효율적 관리를 위한 로컬 데이터베이스(`Room`)까지 안드로이드 프론트엔드 핵심 기술을 종합적으로 학습할 수 있습니다.
3. **인앱(In-App) 편의성 극대화 솔루션**
   * 외부 연동 없이 오직 앱 내부의 완성도 높은 로직과 인터페이스 개선만으로 사용자의 시간 관리 효율을 극대화합니다. 터치 빈도를 최소화하는 UI 설계와 직관적인 시각화 요소를 통해 기존 앱을 한 단계 업그레이드된 **'생산성 특화 솔루션'**으로 변모시키고자 합니다.

## 🗺️ 3. 개발 계획 및 로드맵 (Roadmap)

본 프로젝트는 설계부터 UI 고도화까지 총 4단계에 걸쳐 순차적으로 진행됩니다.

### 📌 Phase 1: 개발 환경 구축 및 기본 UI 레이아웃 설계
* 안드로이드 스튜디오 프로젝트 초기화 및 기본 테마 설정
* 메인 화면 구성: `CalendarView` 및 하단 일정 리스트(`RecyclerView`) 배치
* 일정 등록 커스텀 팝업 창 디자인: 불필요한 단계를 줄이고 제목, 카테고리 색상 선택 스피너(`Spinner`), D-Day 여부 체크박스만 포함한 미니멀 레이아웃 설계

### 📌 Phase 2: 데이터베이스 엔티티 설계 및 로컬 CRUD 구현
* 로컬 데이터베이스 데이터 모델(Entity) 및 데이터 접근 객체(DAO) 설계 (카테고리 컬러 코드 필드 및 D-Day 플래그 포함)
* `Room` 라이브러리를 활용하여 일정을 안전하게 저장하고, 특정 날짜 선택 시 해당 날짜의 일정을 리스트로 불러오는 기본 데이터 동기화 완료

### 📌 Phase 3: 제스처 인식 기반 '초고속 일정 추가' 및 정렬 로직 구현
* `CalendarView` 내 날짜 셀 터치 이벤트 커스텀: 롱 클릭 시 Phase 1에서 설계한 Quick Add 팝업이 즉시 뜨도록 구현
* 마감일(D-Day)로 지정된 일정을 상단에 고정하고 우선순위 순으로 정렬하여 표출하는 `RecyclerView` 어댑터 로직 고도화

### 📌 Phase 4: 인앱 시각화 완성 및 예외 처리 (최종 고도화)
* 달력 메인 화면의 각 날짜 밑에 저장된 일정의 카테고리 색상 도트(Dot)들이 실시간으로 반영되어 찍히는 인앱 시각화 기능 완성
* 일정이 겹치거나 누락되는 상황에 대한 예외 처리 테스트 및 UI 디테일 정돈
* 최종 과제 산출물 레포트 및 시연 영상 준비
