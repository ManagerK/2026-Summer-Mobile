# 📅 Naver Calendar Plus Project (네이버 캘린더 개선 및 클론 코딩)

본 프로젝트는 대학교 **모바일 프로그래밍** 과목의 실습 과제로 진행되는 캘린더 애플리케이션 개발 프로젝트입니다. 기존 '네이버 캘린더' 앱의 핵심 인터페이스를 모방(Clone)하여 구현하되, 기존 앱이 가진 데이터 활용의 한계와 UI/UX적 아쉬움을 보완하여 **사용자 맞춤형 통계 리포트** 및 **시각적 직관성을 극대화한 기능 개선 시스템** 구축을 목표로 합니다.

---

## 📱 1. 대상 앱 소개 및 개선 방향
* **대상 앱:** 네이버 캘린더 (Naver Calendar)
* **핵심 타겟 기능:**
  * 월간 달력 View 인터페이스 및 UI 레이아웃
  * 일정 추가/수정/삭제 (CRUD) 및 카테고리 태깅 기능
  * 날짜별 일정 리스트 조회 및 로컬 저장 (`Room` 라이브러리 활용)
* **주요 개선 항목:**
  1. **데이터 활용 고도화:** 순수 일정 기록을 넘어, 축적된 데이터를 활용한 **'개인화된 시간 관리 통계 및 시각화 리포트 기능'** 추가 (Datalore 연동)
  2. **UI/UX 직관성 개선:** 달력 메인 View에서 카테고리별 지정 색상(Color Tag)이 점(Dot) 형태로 표현되도록 구현하여 일정 분포 인지성 향상
  3. **중요 일정 하이라이트:** 일반 일정과 마감일(D-Day)을 분리하여, 리스트 상단에 중요 마감 일정이 우선 표출되도록 레이아웃 개선

---

## 🎯 2. 앱 선정 이유 및 개선 배경 (Motivation)

1. **기능의 직관성과 UI/UX 개선의 필요성 (Pain Point 분석)**
   * 일상생활에서 가장 대중적으로 사용되는 앱으로 요구사항 분석이 용이하지만, 기존 앱은 일정을 단순히 '텍스트 위주로 기록'하는 데 치중되어 있습니다. 이로 인해 사용자가 자신의 시간 소비 패턴을 한눈에 파악하기 어렵고, 수많은 일정 중 정말 중요한 마감일이나 일정의 성격(공부, 운동 등)을 시각적으로 빠르게 분류하기 어렵다는 아쉬움이 있었습니다.
2. **모바일 프로그래밍 기술 스택의 체계적 훈련**
   * 안드로이드 스튜디오의 핵심 컴포넌트인 `CalendarView` 커스텀, 일정 목록을 유연하게 표현하고 중요도에 따라 정렬하는 `RecyclerView`, 화면 간 데이터 전달(`Intent`), 그리고 데이터 보존 및 쿼리 최적화를 위한 로컬 데이터베이스(`Room`)까지 앱 개발의 필수 요소를 종합적으로 학습하고 다질 수 있습니다.
3. **Datalore 연동을 통한 데이터 분석 기능 도입 (전공 연계 고도화)**
   * 사용자가 등록한 일정 데이터(시간, 카테고리, 빈도 등)를 분석 소스로 활용합니다. 이를 Datalore(Jupyter Hub 기반 파이썬 환경)로 연동하여 사용자의 카테고리별 시간 투자 비율(원형 차트), 요일별 바쁨 지수(막대그래프) 등을 도출함으로써, 기존 앱을 **'지능형 시간 관리 솔루션'으로 개선**하고자 합니다.

## 🗺️ 3. 개발 계획 및 로드맵 (Roadmap)

본 프로젝트는 설계부터 데이터 분석 연동까지 총 4단계에 걸쳐 순차적으로 진행됩니다.

### 📌 Phase 1: 개발 환경 구축 및 UI 레이아웃 설계
* 안드로이드 스튜디오 프로젝트 초기화 및 기본 환경 설정
* 메인 화면 구성: `CalendarView` 및 하단 일정 리스트(`RecyclerView`) 배치
* 일정 등록 화면 구성: 일정 제목, 시간 설정, 카테고리별 고유 컬러(Color Tag)를 선택할 수 있는 스피너(`Spinner`) 및 라디오버튼 디자인
* 중요 일정(D-Day) 지정을 위한 체크박스 인터페이스 추가

### 📌 Phase 2: 데이터베이스 연동 및 로컬 CRUD 구현
* 로컬 데이터베이스 데이터 모델(Entity) 및 데이터 접근 객체(DAO) 설계 (컬러 값 및 D-Day 여부 필드 포함)
* 일정을 데이터베이스에 저장하고, 달력에서 특정 날짜를 선택했을 때 해당 날짜의 일정을 불러오는 연동 로직 완성
* 마감일 일정을 우선순위로 정렬하여 `RecyclerView` 상단에 하이라이트 형태로 표출하는 정렬 로직 구현

### 📌 Phase 3: Datalore 분석 환경 연동 및 데이터 파이프라인 구축
* 로컬 DB에 축적된 일정 데이터를 CSV 또는 JSON 형태로 추출하는 기능 구현
* Datalore(Jupyter 환경)로 데이터를 전달하는 연동 구조 확인 및 데이터프레임(Pandas) 로드 실습

### 📌 Phase 4: 시각화 리포트 구현 및 최종 고도화
* Datalore 파이썬 라이브러리(Matplotlib / Seaborn)를 활용한 카테고리별 시간 소비 리포트 시각화 완성
* 앱 UI 디테일 정돈, 캘린더 내 카테고리별 미니 도트(Dot) 표시 예외 처리 테스트 및 최종 과제 산출물 레포트 작성
